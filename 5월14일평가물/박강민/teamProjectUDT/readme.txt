🚨 우동테 지역선택페이지 맵 구현 과정 (24.5.2 ~ 24.5.13) 🚨

1️⃣ API 선택 - 카카오맵
✔️ api를 사용해 맵 구현
✔️ 스크립트로 클릭시 마커 추가&제거 구현 성공
✔️ 스크립트로 마커 추가&제거 부분 삭제
=> 입력된 데이터값을 불러와서 맵에 구현을 해야 하므로 일반 사용자가 추가&제거 할 수 없어야했다. 
✔️ node.js로 server.js 파일 생성 후 지정포트 설정하여 서버 들어가는 것까지 진행 

2️⃣ SQlite 
✔️ 데이터베이스 저장용으로 SQlite부터 사용
✔️ 스키마 생성 후 데이터 집어 넣기 성공
✔️ SQlite를 사용해 데이터 불러오기 시도 -> 실패

✔️ vscode 확장프로그램으로 SQlite를 설치했으나 사용법에 대한 정보가 명확하지 않아서 다른 DB서비스로 변경
=> 구글링해도 정보가 많이 없어서 시도 끝에 다른 DB서비스 서칭 -> mongoDB 선택

3️⃣ mongoDB
✔️ mongoDB atlas로 무료로 사용 가능하다고 하여 선택
✔️ vscode 확장프로그램 설치 완료
✔️ server.js 파일로 서버 구동 스크립트 작성
=> 대부분 GPT에게 질문하고 코드를 작성했는데 적용이 되질 않음
=> API를 사용할 땐 제공되는 곳의 가이드를 사용해야 된다는 걸 알게됨
✔️ 구글링하다가 카카오맵 말고 네이버맵으로 변경
=> 벤치마킹 플랫폼인 '김캐디'에서 맵 구현으로 네이버맵을 사용하여 변경하게 됨

4️⃣ API 선택 - 네이버맵
✔️ 카카오맵 api와 달리 네이버맵api 는 naver cloud platform 에서 요금을 내고 사용해야 된다고 한다.
=> 하지만 토이프로젝트에서 사용하기에 괜찮은 것이 사용량이 많지 않아서 일정 사용량 한해서 무료로 사용이 가능하다고 함.
✔️ 맵 구현 시도 & 성공 
=> 과정이 카카오api 와 달리 조금 복잡했다.
=> 맵 구현할 때 api키 이슈로 몇번 시도 끝에 성공했다.

✔️ 터미널에서 node.js 실행 -> server.js 파일을 GPT에게 물어보면서 코드 작성을 해서 그런지 제대로 작동을 하지 않음 (구동 됐다가 연결이 끊김)
✔️ 구글링으로 수정하여 구동 성공
=> 하지만 역시나 mongoDB에 저장된 데이터를 불러 올 수 없었다.

🙋🏻 질문내용
 현재 문제점 
- DB에 대한 지식정보가 너무 없다. (본인)-> 계속 시도는 해보겠지만 시간적으로 여유가 없다고 생각이 듦
- 구글링해도 명확하게 설명하는 사람들이 없다.
- GPT 한테 물어보면 상세하게 알려주지만 API 사용하면서 적용하는 문제가 너무 많다.
- DB에 스키마로 테이블 생성하고 수정하고 지우고 다 할 수 있다. 근데 서버구동부터 적용까지가 너무 어렵다.

 이러한 문제들로 몇가지 대안을 추려봤음.
- JS에 지역을 담는 배열을 생성하여 그냥 표출한다. (우선 기본 좌표 찍어두고 청주권내로 표출)
- DB를 어떻게 해서라도 스크립트와 적용해본다. (시간은 좀 많이 걸릴 것 같다. 대안 1번으로 진행하여도 계속 시도할 것임)
- 선생님 조언을 듣고 다른 대안을 생각해본다. ( 팀장님이 SSR[server side rendering] 말씀 하셨는데 이걸로도 가능한 것인가에 대해 물어볼 것임)
- 깔끔하게 지도 포기한다.
